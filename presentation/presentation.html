The Data Foundation

Hunting, Merging, and Cleaning

Building a robust destination database from scratch required rigorous preparation.

  • Sourcing: Hunting down multiple datasets for locations, activities, and coordinates.
  • Merging: Combining disparate CSVs into a unified, relational structure.
  • Cleaning: Handling missing values, standardizing text, and formatting geographic data for spatial relevance.

Data Augmentation with Claude

Enriching the Dataset

Standard datasets lack the nuance required for high-quality LLM context.

  • Used Claude to programmatically generate rich descriptions.
  • Synthesized realistic travel styles and seasonal data.
  • Resulted in a highly diverse, dense dataset perfect for semantic search.

Predicting Travel Styles

Random Forest Classifier

Before the LLM generates prose, we mathematically predict the user's optimal travel style.

  • Trained on our augmented dataset.
  • Evaluates features like Trip Pace, City Scale, and Budget.
  • Provides a grounded, statistical anchor for the LLM to use.

Vector Memory (RAG)

Local Embeddings for Speed

To give the agent instant access to destination knowledge, we built an optimized Retrieval-Augmented Generation pipeline.

  • Local Embedding Models
    Loaded directly into GPU VRAM to avoid API latency.
  • Load-Once Architecture
    Vector database and models are initialized at startup, drastically reducing per-query latency.

Agentic Tools

Giving the LLM Hands

The routing agent (Claude 3.5 Haiku) is equipped with three critical tools it can trigger autonomously:

1. Live Weather

Fetches real-time meteorological data for dynamic itinerary adjustments.

2. Style Classifier

Triggers the Random Forest model to analyze the user's parameters.

3. Destination Search

Queries the local Vector DB to retrieve mathematically relevant locations.

Visibility & Security

LangSmith Tracing

Full observability into the Two-Model architecture:

  • Router (Haiku): Fast, cheap tool selection.
  • Synthesizer (Sonnet): High-quality prose generation.

Data Security

Fully authenticated backend. Chat histories are securely encrypted and tied strictly to user JWT sessions in PostgreSQL.
